## Project Commands

| Command                      | Script                                                                             |
| ---------------------------- | ---------------------------------------------------------------------------------- |
| Install all deps             | `pnpm run install:all`                                                             |
| Dev server (port 3000)       | `pnpm run dev`                                                                     |
| Production build             | `pnpm run build`                                                                   |
| Full validate (gate)         | `pnpm run validate` → type-check:all + lint + format:check + test + functions test |
| Type-check only              | `pnpm run type-check`                                                              |
| Type-check all (+ functions) | `pnpm run type-check:all`                                                          |
| Lint (zero warnings)         | `pnpm run lint`                                                                    |
| Lint fix                     | `pnpm run lint:fix`                                                                |
| Format check                 | `pnpm run format:check`                                                            |
| Format write                 | `pnpm run format`                                                                  |
| Unit tests                   | `pnpm run test`                                                                    |
| Functions tests              | `pnpm -C functions run test`                                                       |
| E2E tests                    | `pnpm run test:e2e`                                                                |
| Build all                    | `pnpm run build:all`                                                               |

**Package manager**: pnpm (not npm). Node 24+ required (v22 prints warnings but works).

## Notes & Gotchas

- `@/` path alias maps to **repo root**, not `src/` — no `src/` directory exists.
- Widget front-face content must use `cqmin` container-query units (never Tailwind `text-sm` etc.).
- `useEffect` should only sync with external systems; computed/derived values go inline.
- Audio context uses a global singleton pattern (see `TimeToolWidget.tsx`).
- `pnpm run validate` runs type-check:all → lint → format:check → test → functions test. All must pass before push.
- Node v22 works but prints "unsupported engine" warnings; CI uses Node 24.
- **Git worktrees & signing server**: The signing binary (`/tmp/code-sign`, configured as `gpg.ssh.program`) identifies the repo by the git process's working directory. Any commit from a worktree path (including `/home/user/spartboard-nightly-*`) returns HTTP 400 "missing source". **Workaround** (confirmed 2026-05-27): create `/tmp/sign-from-root.sh` containing `#!/bin/bash\ncd /home/user/SpartBoard\nexec /tmp/code-sign "$@"`, then commit with `git -c gpg.ssh.program=/tmp/sign-from-root.sh commit --no-verify`. The `--no-verify` is acceptable only after lint-staged has already passed on the same staged files.
- **ESLint from `/tmp/` paths**: Running `pnpm run lint` from a worktree under `/tmp/` produces spurious `Unsafe member access` errors (TypeScript path resolution artifact). Worktrees under `/home/user/` don't have this issue for ESLint itself, but `pnpm run lint` (full repo scan) consistently times out after >3 min. **Workaround**: lint per-directory — `pnpm exec eslint <dir> --max-warnings 0`.
- **Worktree working-directory contamination**: Subagents working in one worktree may leave unstaged changes from other areas (other agents' fixes) in a different worktree's working directory. Before committing from any worktree, always run `git status` and `git restore <foreign-files>` to discard contamination.
- **Verification pattern for orchestrator**: When copying files from a worktree to dev-paul for baseline testing, always backup the ORIGINAL dev-paul file BEFORE the copy (not from the worktree). Pattern: `cp /home/user/SpartBoard/<file> /tmp/<file>.orig` → apply worktree fix → test → `cp /tmp/<file>.orig /home/user/SpartBoard/<file>`. Using `git restore` is even safer after the test. Never use the worktree file as the backup source.
- **Stale-closure risk pattern**: Any widget that creates a `setInterval`/`setTimeout` inside a non-effect callback (e.g. a click handler) and reads props/context values inside that callback is susceptible to stale closures if props can change during the interval. The fix is to assign refs synchronously in the render body (`ref.current = value`) and read from refs inside the callback.
- **`react-hooks/refs` v7 false positives**: `eslint-plugin-react-hooks` v7.0.1 (installed; active via `recommended.rules`) includes `react-hooks/refs: "error"`. Introducing any `setState` call during render (the "adjust state while rendering" pattern) causes this rule to flag all render-body ref assignments in the same component. Silence each affected line with `// eslint-disable-next-line react-hooks/refs -- intentional render-body ref sync (CLAUDE.md pattern)`. Use targeted line disables or a narrow block disable with a matching `/* eslint-enable react-hooks/refs */` immediately after — never a file-tail disable with no closing enable.
- **`GEMINI_SPECIFIC_FEATURES` maintenance**: `adminAnalyticsCompute.ts` uses this array to parse `ai_usage` doc IDs. Any new `genType` added to `index.ts` that writes per-feature docs must also be added here, or analytics will silently misparse the doc ID and drop the feature's data. The 2026-06-03 memory doc note claiming `video-activity-recommend` was already in the array was incorrect — audit notes in memory doc are not a substitute for reading the actual source.
- **Double-handler guard pattern**: When a component registers both a React synthetic event handler and a native window/document listener for the same event (e.g. `onPointerUp` + `window.addEventListener('pointerup')`), event bubbling means both can fire for the same event. Guard with a `committedRef = useRef(false)` reset on the initiating event — first handler to fire sets it `true` synchronously; second handler checks and returns early.
- **Email domain validation**: `str.indexOf('.', atIdx)` searches starting at `atIdx` itself. If `atIdx` is the `@` position and you need a dot _after_ at least one domain character, use `str.indexOf('.', atIdx + 2)`.
- **`data-widget-id` attribute lookup**: This attribute lives on the `.widget` root element (GlassCard). Any keyboard/pointer handler that needs the focused widget's ID must call `element.closest<HTMLElement>('.widget')?.getAttribute('data-widget-id')` — calling `getAttribute` on `document.activeElement` directly always returns null when a child element is focused.
- **`isInput`/`isTypingField` guard must include `'SELECT'`**: Any keyboard handler that needs to pass through native browser interaction (dropdown dismiss, etc.) must include `'SELECT'` in the tag-name check alongside `'INPUT'` and `'TEXTAREA'`. Omitting it causes global shortcuts (Escape → minimize, Delete → confirm-delete) to fire while a native `<select>` dropdown is open. Pattern: `['INPUT', 'TEXTAREA', 'SELECT'].includes(el?.tagName || '') || el?.isContentEditable`.
- **`vi.mock` path resolution in test files**: `vi.mock('./foo')` resolves relative to the test file's directory, not the source file being tested. If the module being tested imports `@/hooks/foo`, the mock must use `vi.mock('@/hooks/foo', ...)` — not `vi.mock('./foo', ...)`. Using the wrong path silently makes the mock inactive (no error thrown); tests may still pass if the real implementation is safe to run in the test environment, but the mock intentions are unfulfilled.
- **Spurious `useEffect` deps pattern**: If a function is listed in a `useEffect` deps array but never called _inside_ the effect body (only in JSX click handlers or other event handlers), it is a spurious dep. If that function's identity changes frequently (e.g. wrapped in `useCallback` that depends on auth state), it will tear down and re-register the effect on every change — classic for paste/keydown listeners. Audit: grep the effect body text for the dep name; if absent, remove it from the array and add a comment explaining why it's intentionally omitted.
- **`CalculatorTool.pressDecimal` pattern**: When a calculator function builds both a display string and a parallel expression string, the expression must be seeded from `newDisplay` when expression is empty (not by appending directly). Appending `'.'` to `""` gives `"."` not `"0."`. The general fix pattern (also used by `pressDigit`): when `prev.expression === ''`, seed with `newDisplay`; otherwise replace the display-length tail of the expression.
- **Testability of unexported hook-internal functions**: When a pure function lives inside a hook (unexported) and needs a real regression test, extract it to the nearest `utils/` file and export it. Inline `normalizeBuggy`/`normalizeFixed` stubs in tests are self-referential — they pass before AND after the production fix and provide no regression protection. Confirmed pattern 2026-06-08: `normalizeSession` → extracted to `utils/videoActivityNormalize.ts` as `normalizeVideoActivitySession` (#1902).
- **LTI null-clobber pattern**: `args.field ?? null` in a Firestore write overwrites a stored value when the field is absent from the launch payload (e.g. privacy-configured relaunches). Prefer `args.field ?? storedDoc.field ?? null` — the seen-section inventory in `nrpsStore.ts` already applied this correctly; the context membership doc path missed it (#1903).
- **`Alt+Arrow` keyboard shortcut guard**: `Alt+ArrowLeft`/`Right` is the standard word-navigation shortcut in text inputs on macOS and Linux. Any global keyboard handler binding these shortcuts must apply the `isTypingField` guard before `e.preventDefault()`. Adding the guard without also moving `preventDefault` inside the guard is a band-aid — the cursor still stalls even if the action doesn't fire.
- **Shallow-copy queue mutation pattern**: `[...queue]` only copies the array shell — element objects are shared references. Mutating `updated[i].status = 'x'` also mutates `queue[i]`. React's state closure captures the pre-render queue reference; a rapid double-click sees the already-mutated state and advances by two instead of one. Fix: `queue.map(item => ({ ...item, newField: value }))` produces a new array with new item objects. Confirmed 2026-06-09: `NextUpWidget.handleNextStudent` (#1914).
- **Normalizer destructuring + `...restData` pattern**: When a hook's `normalizeSession` uses a hand-enumerated literal return, it silently drops every optional field not listed. The correct fix is to destructure ONLY the fields that need explicit normalization logic (guards, defaults, type checks), assign `...restData` for everything else, and spread `restData` first in the return object so explicit fields take precedence. This preserves all future optional fields automatically. Do NOT use a flat `...data` spread — that bypasses the normalization guards entirely. Confirmed 2026-06-09: `normalizeMiniAppSession` (#1912); same pattern as `normalizeVideoActivitySession` (#1902).
- **`window.paste` guard must include `'SELECT'` + `?.` on `e.target`**: Window-level paste event handlers must guard on `SELECT` (in addition to `INPUT`/`TEXTAREA`/`isContentEditable`) to prevent accidental widget insertion when a native dropdown is focused. Also apply `?.` optional chaining when accessing `e.target` properties — synthetic events and detached elements can produce a null target. Pattern: `const target = e.target as HTMLElement | null; if (target?.tagName === 'INPUT' || ... || target?.tagName === 'SELECT' || target?.isContentEditable) return;`. Confirmed 2026-06-09: `Dock.handlePaste` (#1913).
- **`progress === 0` vs phase sentinel**: In animation loops where `progress` resets to 0 at every phase boundary, using `progress === 0` as a quiescent-state guard fires at every transition, not just the initial state. Use a dedicated sentinel value (`phase === 'ready'`, `status === 'idle'`, etc.) for guards that should only fire in the true initial state. Confirmed 2026-06-11: `BreathingWidget` Reset disabled logic (#1934).
- **SSRF redirect guard**: Any `axios` call that precedes a blocklist check must include `maxRedirects: 0`. Without it, axios follows up to 5 redirects by default — a public host can 302 to a private/internal IP and bypass the blocklist entirely. Pattern already applied to `fetchExternalProxy`; extended to `checkUrlCompatibility` in #1937. Any future Cloud Function that fetches a user-supplied URL must include this flag.
- **Verbatim-EN placeholder in i18n locales**: A translation key can be "present" in a locale file but still serve as an untranslated placeholder if its value was copied from the EN source string. `i18next.defaultValue` triggers only when the key is absent — a wrong stored value renders verbatim. Test assertion pattern: `expect(locale.ns.key).not.toBe(en.ns.key)` (not just `toHaveProperty`). Confirmed 2026-06-11: `boardBreadcrumb.openManager` and `collectionSwitcher.title` in DE/ES/FR (#1936).
- **FP noise in Firestore persistence**: When a computed value passed to Firestore is the result of floating-point arithmetic (e.g. `min + i * step`), round before storing with `parseFloat(value.toFixed(N))` where N preserves the needed decimal precision. Rounding at display-time only masks the bug — the dirty value is still stored and rendered verbatim in settings panels. Confirmed 2026-06-12: `NumberLine/Widget.tsx` `addMarker()` (#1947).
- **Partial-credit `isCorrect` invariant**: In partial-credit grading (`isCorrect`, `pointsEarned`, `pointsMax` triple), `isCorrect` must be derived as `pointsEarned >= pointsMax` — never from a stricter formula (e.g. `givenPairs.length === total`). Mixing a strict correctness formula with a lenient scoring formula produces `isCorrect=false` when `pointsEarned=pointsMax`, breaking scoreboard and streak logic. Confirmed 2026-06-12: `useQuizSession.ts` `gradeAnswer()` matching partial-credit path (#1950).
- **`e.key` case-sensitivity for keyboard shortcuts**: `e.key` is case-sensitive. With CapsLock active, letter keys produce uppercase (`'P'` not `'p'`). Any global keyboard handler that compares `e.key === 'x'` must use `e.key.toLowerCase() === 'x'` instead. The widget-level `DraggableWindow` handler already used toLowerCase; the global `DashboardView` path missed it for Alt+P (fixed #1956). Confirm all shortcut comparisons normalise to lowercase.
- **`.toISOString()` timezone pitfall for local-date comparisons**: `new Date(localMidnightTimestamp).toISOString().split('T')[0]` shifts to UTC before formatting. For users in UTC+ timezones, local midnight converts to the previous day in UTC (e.g. 2026-06-13 00:00:00 UTC+12 becomes 2026-06-12T12:00:00Z). Use local time methods: `${d.getFullYear()}-${String(d.getMonth()+1).padStart(2,'0')}-${String(d.getDate()).padStart(2,'0')}`. Confirmed 2026-06-13: `CalendarWidget.isBlocked` (#1955).
- **`normalizeActivityWallLibraryEntry` pattern**: Same hand-enumerated literal return as #1902/#1912. Extracts to `utils/activityWallNormalize.ts` with `...restData` spread (fixed #1960). Both remaining audit targets (`useQuizSession.ts` and `useGuidedLearningSession.ts`) confirmed clean 2026-06-14 — neither has a hand-enumerated normalizer.
- **Stale `onBlur` on unmounting controlled input**: When a state setter unmounts a focused input (e.g. `setIsEditingTitle(false)`), React batches the update and the input fires a synchronous `blur` event during unmount — before the new state is committed. The `onBlur` callback at that moment is the stale one from the previous render, which closes over old state values. Guard with an `isCancellingRef = useRef(false)` set synchronously before the state setter; check at the top of the save handler and short-circuit if true. Confirmed 2026-06-14: `DraggableWindow` custom title Escape path (#1965).
- **Verbatim-EN collection UI placeholders**: When the Collections / Boards Modal feature was shipped, DE/ES/FR locale files received copy-pasted English source strings as placeholders. These are "present" keys that never trigger the `defaultValue` fallback — non-EN teachers see English silently. Pattern: search for blocks of locale keys where `de.value === en.value` across an entire namespace, not just individual keys. Fixed 59 such keys in #1964. A second pass fixed 10 more `shareCollection` keys in #1977 — namespaces can accumulate new verbatim-EN values across feature branches even after an initial fix.
- **Stale-closure regression test pattern for Escape-cancel**: Subagent regression tests for the stale `onBlur`/Escape bug must use `act(() => { fireEvent.keyDown(input, { key: 'Escape' }); fireEvent.blur(input); })` — wrapping BOTH events inside the same `act()` call. jsdom does NOT fire `blur` automatically when a DOM node is removed; without the manual `blur`, the test passes on BOTH original and fixed code (the race is never reproduced). The single `act()` defers React's state flush until after both events have fired, replicating the browser's synchronous blur-during-unmount sequence. Confirmed required in 2026-06-15 orchestrator review of #1974 and #1975 subagent tests — both agents wrote bare `fireEvent.keyDown(Escape)` without the blur, producing invalid tests that passed before the fix.
- **`proportionsValid` must include value-range checks, not just `isFinite`**: Any guard that validates whether layout proportion fields (`xProp`, `yProp`, `wProp`, `hProp`) are already in proportional form must check BOTH `Number.isFinite(v)` AND `Math.abs(v) <= 1.5` (or appropriate bounds). Pixel values like `300` pass `isFinite` — a partial-migration widget with pixel `xProp`/`yProp` but valid proportional `wProp`/`hProp` would satisfy an `isFinite`-only guard and skip migration, leaving position fields as raw pixels. Confirmed 2026-06-15: `migrateWidgetToProportional` (#1976).

- **`vi.mock()` stubs for modules with module-level side effects**: If a source file executes code at import time (e.g. `onDocumentWritten(...)`, `setGlobalOptions(...)`, `initializeApp()`), any test file that imports it (directly or transitively) must provide `vi.mock()` stubs for all side-effecting modules BEFORE the import. Without stubs, Vitest crashes with `ERR_MODULE_NOT_FOUND` or throws during module initialisation — silently dropping all tests in the file (no failure, just omission). The fix mirrors the sibling test files that already use the correct hoisting pattern. Confirmed 2026-06-22: `mirrorPlcIndex.test.ts` (#2047).
- **`toISOString()` launch-key scope**: Any key computed as `new Date().toISOString().slice(0, 10)` for scheduling/launch deduplication will use the UTC date, diverging from local midnight for UTC+ users. Use `getTodayStr()` (which wraps `getFullYear/getMonth/getDate`) for any "today" marker that governs teacher-facing scheduling logic. Confirmed 2026-06-22: `ScheduleWidget.checkAutoLaunch` (#2046).
- **Enter double-commit on title rename**: Same class as the Escape-cancel stale-onBlur pattern, but triggered by the commit path. Pressing Enter calls `saveTitle()` directly, which schedules `setIsEditingTitle(false)`. Before React commits the unmount, the browser fires a synchronous `blur` — calling `saveTitle()` a second time via `onBlur`. `isConnected` does NOT work here (node is still connected at blur time). Fix: `hasCommittedTitleRef = useRef(false)` guard, set to `true` on first commit, reset in the render body while editing is active (`if (isEditingTitle) hasCommittedTitleRef.current = false`). React 18 automatic batching ensures `updateWidget`'s state update and `setIsEditingTitle(false)` commit in a single render cycle — no intermediate render resets the guard before blur fires. Confirmed 2026-06-22: `DraggableWindow` (#2045).
- **CSS source-order wins cascade over `@tailwind base` (equal-specificity rules)**: A manual `@media` block placed AFTER `@tailwind base` in `index.css` wins the cascade over Tailwind-generated rules of equal specificity — CSS source order is the tie-breaker when specificity is the same. The `tailwind.config.js` `reducedMotionPlugin` injected into `@tailwind base` does NOT override a later `@media (prefers-reduced-motion: reduce)` block in the same file, because both produce identically-specific `@media { .class { animation: none !important } }` declarations. When hand-editing `index.css` reduced-motion blocks, verify the intended set of suppressed classes matches what `tailwind.config.js` defines as non-essential. Confirmed 2026-06-24: `.animate-spin` frozen for reduced-motion users (#2068).
- **WCAG 2.3.3 — essential animations must be preserved under reduced-motion**: Loading spinners (`animate-spin`, `Loader2`, etc.) are essential animations: freezing them removes the only in-progress affordance during async operations, which reads as "stalled" to the user. Under WCAG SC 2.3.3, animations that communicate necessary state information (loading, progress) must NOT be suppressed by `prefers-reduced-motion` rules. Only decorative/looping animations (slow spins, jitter, pulse, bounce) should be suppressed. `tailwind.config.js`'s `reducedMotionPlugin` intentionally excludes `animate-spin` from the suppression list; any `index.css` manual block must mirror this exclusion. Confirmed 2026-06-24: (#2068).
- **`quizDriveService` stats-block dedup pattern**: `exportResultsToSheet` and `buildResultsSheetDataShared` are two separate code paths that each do their own dedup. Drive-sync/arrayUnion races can write duplicate question IDs; both paths must apply a `Set<string>` fence before iterating `questions`. The response-level stats loops (`answeredSet`/`correctSet`) only dedup per-student responses — they do not protect against duplicate question rows in the output. Dedup must happen at the `questions` level before any loop that emits one row per question. Confirmed 2026-06-24: `exportResultsToSheet` stats block (#2065).

- **jsdom programmatic blur is always a no-op in tests**: `e.currentTarget.blur()` called inside a `keyDown` handler does NOT dispatch a real `blur` event in jsdom — it is silently ignored regardless of whether the element was focused via `fireEvent.focus()`.
